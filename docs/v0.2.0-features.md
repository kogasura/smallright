# 設計書: smallright MCP サーバー v0.2.0

## 概要

ブラウザ自動化 MCP サーバー smallright に、以下の 3 機能を追加する。

1. `evaluate` ツールの async/await 対応 — スクリプト内で `await` が使えない制限を解消する
2. `upload_file` ツール新設 — `input[type=file]` にファイルをセットする
3. `download_file` ツール新設 — ボタンクリック起因のファイルダウンロードを検証する

## 技術スタック

- TypeScript / Node.js（既存スタックをそのまま踏襲）
- Playwright（既存の `Services.browser` 経由で `Page` を取得）
- MCP SDK（`@modelcontextprotocol/sdk` — 既存の `server.tool()` 登録パターンを踏襲）
- Zod（パラメータスキーマ定義、既存パターンを踏襲）

## アーキテクチャ

既存のレイヤー構成を維持する。

```
src/
  tools/
    evaluate.ts          <- 修正
    upload-file.ts       <- 新規
    download-file.ts     <- 新規
  server.ts              <- ツール登録を追加
  types.ts               <- 変更なし（BatchStep 拡張は v0.3.0 スコープ）
```

`upload_file` / `download_file` はいずれも単方向操作（スナップショット差分なし）であるため、`StateDiff` を返さず、操作結果の構造体を JSON で直接返す設計とする。これは既存の `evaluate` / `screenshot` / `read_table` が採用しているパターンと一致する。

## データモデル

### DownloadResult（download_file の戻り値）

```typescript
interface DownloadResult {
  filename: string;        // download.suggestedFilename()
  size: number;            // バイト数（fs.stat 経由）
  mimeType: string | null; // 拡張子からインラインマップで推定
  preview?: string;        // テキストファイルの場合のみ先頭 20 行
}
```

`mimeType` は Playwright の Download API が直接 MIME を提供しないため、ファイル名の拡張子から推定する。主要拡張子（csv, json, pdf, txt, xlsx 等）をインラインマップで対応し、パッケージ依存を追加しない。

## API 設計（ツール仕様）

### evaluate（修正後）

```
evaluate(script: string) => string (JSON)
```

- 変更点: スクリプトを async IIFE 文字列でラップして `page.evaluate` に渡す
  ```typescript
  const wrapped = `(async () => { ${params.script} })()`;
  const result = await page.evaluate(wrapped);
  ```
- インターフェース（パラメータ・戻り値）は変更なし

### upload_file（新規）

```
upload_file(paths: string[], label?: string, selector?: string) => string (JSON)
```

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| paths | string[] | Yes | アップロードするファイルの絶対パス配列 |
| label | string | No | ラベルテキストで input[type=file] を特定 |
| selector | string | No | CSS セレクターで直接指定 |

優先順位: `label` > `selector` > ページ内最初の `input[type=file]`

`input[type=file]` は hidden 要素であるため scan に含まれない。`scan()` や `Services.elements` は使用せず、Playwright の Locator API で直接特定する。

- `label` 指定: `page.getByLabel(label).setInputFiles(paths)`
- `selector` 指定: `page.locator(selector).setInputFiles(paths)`
- どちらも未指定: `page.locator('input[type=file]').first().setInputFiles(paths)`

`setInputFiles` 前に各パスを `fs.access` で存在確認し、アクセス不可の場合はエラーをスローする。

- 戻り値: `{ success: true, paths: string[] }` の JSON

### download_file（新規）

```
download_file(text: string, role?: string, timeout?: number) => string (JSON)
```

| パラメータ | 型 | 必須 | 説明 |
|---|---|---|---|
| text | string | Yes | クリック対象要素のテキスト |
| role | string | No | 要素のロール（click ツールと同じ意味） |
| timeout | number | No | ダウンロード待機タイムアウト ms（デフォルト 30000） |
| save_path | string | No | 指定時は `download.saveAs(savePath)` で保存。未指定時は `download.delete()` で削除 |

内部動作:
1. `page.waitForEvent('download', { timeout })` の Promise を先行して生成
2. `s.elements.resolveByText(text, elements, undefined, undefined, role)` で要素を解決
3. `resolveLocator(page, resolved).click()` でクリック
4. download イベントを await で受信
5. `download.suggestedFilename()` でファイル名取得
6. `download.path()` で一時ファイルパス取得 -> `fs.stat` でサイズ確認
7. テキストファイル（.txt / .csv / .json / .log）なら `createReadStream` + readline で先頭 20 行のみ読み取り `preview` として格納（大きなファイルの全量読み込みを避けるため）
8. `save_path` 指定時は `download.saveAs(savePath)` で保存。未指定時は `download.delete()` で一時ファイルをクリーンアップ
9. `DownloadResult` を JSON で返す

エラーケース:
- 要素が見つからない場合: `throw new Error` — `wrap()` がキャッチして isError レスポンスを返す
- AmbiguousMatch の場合: 候補リストを JSON で返す（click ツールと同じ挙動）
- タイムアウト: Playwright が例外を投げ、`wrap()` がキャッチする

## リスク・注意点

- **`download.path()` が null を返すケース**: Playwright はダウンロード失敗時に `null` を返す。明示的な null チェックと分かりやすいエラーメッセージを実装すること。
- **`upload_file` の要素特定**: `input[type=file]` は hidden のため scan に含まれない。`page.getByLabel` / `page.locator` で直接特定する方針のため、label テキストと実際の `<label for="...">` 対応が正しく設定されていることが前提となる。
- **BatchStep への upload 追加**: `types.ts` の `BatchStep` union への `"upload"` 追加および `paths` フィールド追加は v0.2.0 スコープ外。v0.3.0 で batch 対応と合わせて実装する。
