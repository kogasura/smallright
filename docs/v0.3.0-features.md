# 設計書: smallright MCP サーバー v0.3.0

## 概要

ブラウザ自動化 MCP サーバー smallright に、以下の機能を追加する。

1. `set_viewport` ツール — ビューポートサイズをプリセットまたは任意サイズに変更する
2. `navigate_back` ツール — ブラウザの「戻る」操作を実行する
3. `press_key` ツール — キーボードキーを送信する（Tab, Enter, Escape, ArrowDown 等）
4. `wait_for` ツール — 指定テキストまたはセレクターが出現するまで待機する
5. `hover` ツール — 要素にマウスホバーする
6. vitest ユニットテスト — 純粋関数を対象にしたユニットテストを整備する
7. GitHub Actions CI — PR 時に型チェックとテストを自動実行する

## 技術スタック

- TypeScript / Node.js（既存スタックをそのまま踏襲）
- Playwright（既存の `Services.browser` 経由で `Page` を取得）
- MCP SDK（`@modelcontextprotocol/sdk` — 既存の `server.tool()` 登録パターンを踏襲）
- Zod（パラメータスキーマ定義、既存パターンを踏襲）
- vitest（新規追加 — devDependencies）

### 戻り値の設計方針（既存パターンを踏襲）

| パターン | 戻り値 | 採用ツール |
|---|---|---|
| ページ遷移が発生する操作 | `ActionModeState` | navigate, set_viewport, navigate_back, wait_for |
| DOM 変化を伴うが遷移しない操作 | `StateDiff` | click, fill, press_key, hover |

## データモデル

新規のデータ型はなし。既存の `ActionModeState`、`StateDiff`、`AmbiguousMatch` をそのまま使用する。

## API 設計（ツール仕様）

### set_viewport（新規）

パラメータ: `width?: number`, `height?: number`, `preset?: "mobile" | "tablet" | "desktop"`

プリセット値:
- mobile: 375x812
- tablet: 768x1024
- desktop: 1280x720

- `preset` が指定された場合はプリセット値を使用し、`width`/`height` で上書き可能
- どちらも指定なしの場合はエラーをスロー
- `page.setViewportSize({ width, height })` を呼び出し
- `waitForSpaReady` + `elements.scan` + `state.buildActionModeState` して返却（navigate.ts と同パターン）

### navigate_back（新規）

パラメータ: なし

- `page.goBack({ waitUntil: "domcontentloaded", timeout: 30000 })` を呼び出し
- goBack が null を返す場合（遷移先がない場合）はエラーをスロー
- navigate.ts と同パターンで `waitForSpaReady` + `scan` + `buildActionModeState` して返却
- プロファイル読み込みは実施しない（既にロード済みのため不要）

### press_key（新規）

パラメータ: `key: string`（例: "Tab", "Enter", "Escape", "ArrowDown"）

- `snapshotBefore` を取得
- `page.keyboard.press(key)` を呼び出し
- `waitForTimeout(300)` で DOM 安定化
- `snapshotAfter` を取得して `computeDiff` し StateDiff を返却（click.ts と同パターン）

### wait_for（新規）

パラメータ: `text?: string`, `selector?: string`, `timeout?: number`（デフォルト 10000ms）

- `text` 指定: `page.getByText(text).first().waitFor({ state: "visible", timeout })`
- `selector` 指定: `page.locator(selector).first().waitFor({ state: "visible", timeout })`
- `text` と `selector` が両方ない場合はエラーをスロー
- 成功後に `scan` + `buildActionModeState` して返却（待機完了後のページ状態を返す）

### hover（新規）

パラメータ: `text: string`, `role?: string`, `zone?: string`, `index?: number`

- click.ts と同構造で `resolveByText` → `resolveLocator` → `locator.hover({ timeout: 10000 })` を呼び出し
- `waitForURL` ブロックは不要（hover は遷移を起こさない前提）
- `waitForTimeout(300)` で DOM 安定化
- StateDiff を返却

```typescript
const PRESETS = {
  mobile:  { width: 375,  height: 812  },
  tablet:  { width: 768,  height: 1024 },
  desktop: { width: 1280, height: 720  },
};
const base = params.preset ? PRESETS[params.preset] : undefined;

const height = params.height ?? base?.height;
if (!width || !height) throw new Error("width/height または preset を指定してください");
const page = await s.browser.getPage();
await page.setViewportSize({ width, height });
// navigate.ts と同パターンで waitForSpaReady → scan → buildActionModeState → dialogs 合成
```

**element-registry.test.ts** — `createElementRegistry()` の `resolveByText`、`resolveByLabel` を検証。`scan` は `page.evaluate` 依存のためテスト対象外。

| テストケース | 検証内容 |
|---|---|
| exact match が prefix より優先される | 完全一致候補が返る |
| 単一候補は InteractiveElement を直接返す | AmbiguousMatch にならない |
| 複数候補は AmbiguousMatch を返す | candidates 配列を含む |
| index 指定で候補を選択できる | 指定インデックスの要素が返る |
| 一致なしは null を返す | 戻り値が null |
| resolveByLabel: label フィールドが最優先 | text より label が優先される |
| resolveByLabel: placeholder フォールバック | label なし・text なしでも placeholder で解決 |

**state-differ.test.ts** — `computeDiff` を検証（`takeSnapshot` は `page.evaluate` 依存のため除外）。

| テストケース | 検証内容 |
|---|---|
| URL が変化した場合 | url.changed === true、from/to が正しい |
| URL が変化しない場合 | url.changed === false |
| コンテンツが変化したゾーン | changedZones に含まれる |
| コンテンツが同一のゾーン | unchangedZones に名前が含まれる |
| before にあり after にないゾーン | changedZones に空コンテンツで含まれる |

**state-builder.test.ts** — export 追加後の `collapseSimilar`、`classifyTextPattern` を直接検証。

| テストケース | 検証内容 |
|---|---|
| 10件未満はそのまま返す | 入力と同数の要素 |
| 10件以上の同一グループを折りたたむ | 3件 + "...and N more similar elements" |
| 異なるグループは独立して折りたたまれる | グループ別に 3+1 構造 |
| 日付文字列は "repetitive" を返す | "2026年4月8日" など |
| 通常テキストは "unique:text" を返す | "ログイン" → "unique:ログイン" |
| 空文字列は "repetitive" を返す | "" → "repetitive" |

## リスク・注意点

- **`page.goBack()` の null 戻り値**: ブラウザ履歴がない場合に null を返す。明示的に検出してエラーをスローすること。
- **`set_viewport` 後のレンダリング遅延**: `page.setViewportSize()` はビューポートを即時変更するが、レスポンシブなフレームワークは CSS ブレークポイント切り替えに数百 ms かかる場合がある。`waitForSpaReady` で吸収できるが、フレームワーク依存で不十分なケースは既知の制限として許容する。
- **`collapseSimilar` / `classifyTextPattern` の export 追加**: `StateBuilder` インターフェースへの影響はないが、パブリック API として露出することになる。テスト専用の内部関数であることを JSDoc コメントで明記することを推奨する。
- **CI で playwright ブラウザ不要**: 今回のユニットテスト対象は全て純粋関数のため `npx playwright install` は不要。将来的に E2E テストを追加する場合は別ジョブに分離すること。
- **package.json の version が `0.1.0`**: `server.ts` が既に `"0.2.0"` を使用しており不一致が生じている。単位7 で両方を `0.3.0` に統一する。
